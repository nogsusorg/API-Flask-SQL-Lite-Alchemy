Endpoints Disponibles (Protegidos)

Todas las rutas se acceden a través de la base /api.

  • GET /api/products?page=1&per_page=5

    Descripción: Recupera un listado paginado de productos. Los parámetros page y per_page son opcionales.

    Retorno: JSON con lista de productos y metadatos de paginación.

  • GET /api/products/<int:product_id>

    Descripción: Recupera los detalles de un producto específico utilizando su ID.

    Retorno: JSON del objeto producto o 404 si no existe.

  • POST /api/products

    Descripción: Crea un nuevo producto. Requiere un cuerpo JSON con name, price y opcionalmente description.

    Ejemplo JSON: {"name": "Tablet Pro", "price": 899.99, "description": "10 pulgadas, 128GB"}

    Retorno: JSON con el ID del producto creado (código 201).

  • DELETE /api/products/<int:product_id>

    Descripción: Elimina un producto específico utilizando su ID.

    Retorno: JSON de confirmación o 404 si no existe.
